Error: "features": required field is not set

Terraform - “Error: “features”: required field is not set”

Issue:

Something I came across time and time again when working through the MS Learn Course Automate your deployments with Azure DevOps was the following error in my Azure DevOps build pipelines:

"Error: "features": required field is not set"

## Steps to reproduce:

Create a basic terraform configuration using terraform 0.12 or later and AzureRM provider version 2.0 or later. E.g:

# Configure the terraform block
terraform {
  required_version = "> 0.12.0"
}

# Configure the Azure Provider
provider "azurerm" {
  version = ">=2.0.0"
}

# Create a new resource group
resource "azurerm_resource_group" "rg" {
  name     = "myTFResourceGroup"
  location = "uksouth"

  tags = {
    environment = "TF sandbox"
  }
}

and run:

terraform plan

Resolution:

Add an empty features block to the provider block in your root module:

provider "azurerm" {
  version = ">=2.0.0"
  features {}
}

Further details:

As of AzureRM Provider 2.0.0 - (February 24, 2020):

“Provider Block: The Azure Provider now requires that a features block is specified within the Provider block, which can be used to alter the behaviour of certain resources - more information on the features block can be found in the documentation”.

“features - (Required) A features block as defined below which can be used to customize the behaviour of certain Azure Provider resources.”
